**네트워크 통신 방식
	1)TCP방식
		상대방 컴퓨터와 연결을 먼저 맺은 후, 그 연결된 회선을 통해 데이터를 주고 받는 방식
		
	2)UDP방식
		상대방 컴퓨터와 연결을 먼저 맺지 않고, 그냥 데이터를 막 보내는 방식
		
		연결이 되어 있지 않기 때문에,
		데이터를 보내는 시점에 보내고자 하는 데이터 뿐만아니라, 목적지 주소와 데이터를 하나의 세트로 만들어야한다.
		-> "패킷"
		
		UDP방식의 데이터를 전송하고 받기 위해서는
		DatagramSocket클래스의 send메소드와 receive 메소드를 이용한다.
		이때,
		send메소드와 receive메소드는 DatagramPacket을 매개변수로 전달해야한다.
		
		데이터를 보낼 때도 패킷에 담아 보내야하고,
		데이터를 받을 때도 패킷에 담아와야한다.
		receive(DatagramPacket p)
		send(DatagramPacket p)
		
		서버측 패킷 생성
		DatagramPacket(byte[] buf, int length, InetAddress address, int port)
		
		패킷 생성시 -- 클라이언트
		DatagramPacket(byte[] buf, int length)
		
		메소드
		InetAddress	getAddress()

CUI
Character User Interface

컴퓨터에게 일을 시킬 때, 문자로 시키는 방식


GUI
Graphic User Interface
프로그램 사용을 그래픽화면을 통해 사용하는 방식
컴퓨터에게 일을 시킬 때,
그래픽화면의 버튼을 누르는 등으로 일을 시키는 방식

배치관리자(LayoutManager)
	컴포넌트들을 화면에 배치하기 위한 클래스들

	1) FlowLayout
		물흐르듯이 컴포넌트들을 왼쪽에서 오른쪽으로, 위에서 아래로 자동으로 배치하는 방식
		
	2) GridLayout
		화면을 행과 열로 분할하여 컴포넌트들을 배치하는 방식
		행부터 채워져서 들어간다.
		
	3) BorderLayout
		화면을 위, 아래, 좌, 우, 가운데 
		총 5개의 영역으로 분할하여 배치하는 방식
		
		컴포넌트의 수에 따라 위,아래,좌,우 는 생략할 수 있지만
		가운데는 생략할 수 없다.
		가운데에 중요한 구성요소를 두고, 그것을 중심으로
		덜 중요한 구성요소들을 위,아래,좌,우에 배치하는 방식

--> 복잡한 화면을 구성할 땐
	프레임 하나만으로는 화면을 구성하기가 어렵다.
	이럴 때, 서브컨테이너 역할을 하는 패널을 이용한다.
	
--> 패널을 기본 레이아웃방식이 FlowLayout이고,
	프레임은 기본 레이아웃방식이 BorderLayout임

	BorderLayout에서 컴포넌트를 담을 때,
	위치를 지정하지 않으면 기본값은 CENTER
	센터에 담고 센터에 담으면 맨 마지막 센터에 담은 컴포넌트가 보인다.
